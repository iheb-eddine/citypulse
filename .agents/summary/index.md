## Architecture Summary

CityPulse is a **FastAPI monolith** where `app/main.py` contains all routes and business logic, with `classifier.py` and `news.py` extracted as modules for external API isolation. It uses **SQLite** with a single `reports` table, **Groq API** for AI (vision + text), **DBSCAN** for spatial clustering, and **Folium** for server-side map generation. The app gracefully degrades when AI services are unavailable — no feature causes a 500 error due to external service failure.

## Key Patterns

- **Fallback-first AI integration:** Every AI call has a fallback path; the app never blocks on AI failures
- **Server-side rendering:** Maps generated by Folium on the server, not client-side JS mapping
- **Monolith with module extraction:** Core logic in one file, external API wrappers extracted
- **Magic-byte file validation:** File types checked via binary headers, not extensions
- **EXIF stripping:** All uploaded photos have metadata removed for privacy
- **SSE for real-time:** Server-Sent Events push new reports to connected dashboards
